Summary of Requirements

Number Requirement Reason
1 Solution must be fully automated The cost of working off-site, during an expensive window of development, makes it impractical to depend on manual steps for manipulating data. The cost of manual work would be repeated if the results had to be regenerated.
2 Solution must support substitution of real data with fake data sets Because of information governance concerns, much of the input data that could be used to run the protocol cannot be taken off-site. In order to allow substantial development and testing of the protocol to be done in their normal work environment, developers would have to be able to simulate sensitive data with fake data.
3 Use technologies that make a complete separation between data and code. Some technologies such as spreadsheets save macros together with the data to which they are applied. If the data are sensitive and cannot be taken off-site, and if the code can't be separated from the data, then the code cannot be taken off-site as well.
4 Quantify data cleaning through sensitivity variables. The protocol must audit data cleaning via sensitivity variables that scientists can use to assess the effects of changes on subsets of results.
